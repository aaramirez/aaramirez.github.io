Ionic es un framework situado en la capa de presentación que permite desarrollar aplicaciones móviles multiplataforma para iOS y Android. Los desarrolladores del framework se han enfocado en mejorar el desempeño de las aplicaciones móviles construidas con tecnología web.

Bienvenido al Curso - Componentes de Ionic - CSS, Este curso se enfoca en mostrar todos los componentes CSS del framework y cómo se pueden utilizar para construir plantillas (templates). Los componentes de Ionic - CCS son una serie de directivas (En terminología de Angular) que se pueden utilizar como clases y algunas como etiquetas (tags). La idea es repasarlas e imaginar diferentes casos de uso.

Herramientas de trabajo

Vamos a utilizar play.ionic.io y Codepen.io para mostrar el resultado del trabajo de forma ágil sin necesidad de crear proyectos nuevos en la línea de comandos de Ionic. Luego estas plantillas se pueden utilizar en proyectos reales de aplicaciones móviles con Ionic.

Con play.ionic.io y Codepen.io tenemos la posibilidad concentrarnos en trabajar en el markup HTML y CSS de las plantillas y ver el resultado inmediatamente sin ciclos de compilación (build). Esto acelera el proceso de generación de las plantillas y además permite compartir el resultado del trabajo inmediatamente, sin la complejidad de generar una aplicación que luego se debe distribuir.

play.ionic.io

En play.ionic.io hay tres pestañas: HTML, CSS y JS. Además cuatro botones: SAVE, FORK, PREVIEW y NEW.

En cada pestaña se encuentra el código HTML, CSS o Javascript de la aplicación. La aplicación está compuesa sólo por estos tres segmentos de código, más las librerías o recursos a los cuales se haga referencia desde esas tres ventanas. Es decir, se pueden incluir recursos que están disponibles en la red.

Los cuatro botones permiten:

  • SAVE: Salvar el trabajo. Esto no genera un archivo, sencillamente se salva y es accesible a través del URL. Si desea acceder al trabajo realizado o compartirlo sólo debe enviar el URL o guardarlo. El URL es de la forma (http://play.ionic.io/app/ceba5803d714). Cuando se salva se utiliza un identificador que permite hacer referencia de nuevo al código salvado.
  • FORK: Permite crear una nueva aplicación partiendo del código actual. De hecho cada vez que compartamos un proyecto, lo primero que hay que hacer el FORK para generar una nueva aplicación y modificarla. El resultado se puede salvar para poder acceder al código luego. Para poder ver el trabajo realizado sencillamente se copia el URL.
  • PREVIEW: Muestra el resultado del trabajo en una ventana independiente.
  • NEW: Crea un nuevo proyecto e inicia desde el principio. Este botón genera un nuevo código de la aplicación. Al salvar se debe guardar el URL para poder recuperar la aplicación o compartirla.


Codepen.io

Por su parte Codepen.io también nos permite desarrollar y probar ideas basadas en tecnología Web. También cuenta con tres tabs principales donde colocamos el código HTML, CSS y JS, además de una ventana de preview. La buena noticia es que en Codepen.io podemos conseguir infinidad de ejemplos ya hechos que pueden servir de inspiración para hacer nuestros propios experimentos.


En Codepen.io vamos a publicar los ejemplos o ejercicios resueltos y los vamos a mostrar en un teléfono para dar una idea de cómo es el resultado y su interacción.

En este caso estamos mostrando a través de Codepen.io un ejemplo de una aplicación con un menú de tipo retractil o drawer.

Se puede interactuar con el teléfono y ver el código HTML, CSS o JS presionando las pestañas que aparecen en la parte superior, o se puede ver el resultado en la pestaña Result. Adicionalmente, aunque no es muy cómodo se puede editar el código y hacer ajustes.

Para realizar cambios y pruebas recomendamos en su lugar dirigirse a Codepen.io y lo invitamos a crear sus propios Pen y compartirlos.
